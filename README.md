# module4_is601# Module 4: Object-Oriented Basics Part 1

## Module Overview
This module introduces students to the fundamental concepts of Object-Oriented Programming (OOP) in Python. Students will learn about classes, objects, methods, attributes, encapsulation, inheritance, and polymorphism. They will begin refactoring their previous CSV processor command-line application using basic OOP techniques to enhance code organization and reusability.

### Why Object-Oriented Programming?
OOP is a powerful paradigm that allows for more organized, modular, and maintainable code. It provides a foundation for creating complex systems by promoting code reuse, data encapsulation, and logical structuring of code. Understanding OOP basics is crucial for developing scalable and efficient Python applications.

### Learning Outcomes
- Implement object-oriented programming principles in Python.
- Apply professional program techniques such as DRY and Error handling using LBYL and EAFP.
- Create and manipulate CSV files using Python.

### Submit
**Activity Type:** Basic OOP Refactoring Assignment  
**Activity Title:** Refactoring CSV Processor with Basic OOP Concepts  
**Grading Type:** Points  
**Submission Instructions:** Submit a link to your GitHub repository containing the partially refactored project.  
**Instructions:**
1. Fork your previous CSV Processor Command-Line Application repository.
2. Begin refactoring the application using basic OOP principles:
   - Create at least two classes for major components (e.g., CSVReader, CSVWriter)
   - Implement proper encapsulation using private and public attributes
   - Create instance methods for key functionalities
   - Implement at least one class method and one static method where appropriate
   - Create a simple inheritance relationship (e.g., a BaseReader class and a CSVReader subclass)
   - Demonstrate polymorphism through method overriding in your inherited class
3. Update some of your unit tests to work with the new OOP structure
4. Update your documentation to reflect the new OOP design
5. Ensure your GitHub Actions still run successfully with the refactored code

**Grading Expectations:**
- Proper implementation of basic OOP concepts (classes, objects, methods, attributes)
- Correct use of encapsulation (private and public attributes)
- Appropriate use of instance methods, class methods, and static methods
- Implementation of inheritance and polymorphism
- Updated and passing unit tests for the refactored components
- Clear documentation of the basic OOP design
- Successful GitHub Actions runs

**Alignment:**
- Implement object-oriented programming principles in Python.
- Apply professional program techniques such as DRY and Error handling using LBYL and EAFP.
- Create and manipulate CSV files using Python.
